# 50 — 메모리 프로토콜

이 프로젝트는 `.claude/memory/` 아래에 영구적이고 버전 관리되는 메모리를 둡니다.
이 하네스의 핵심 가치 — **자신의 실수를 반복하지 않는다** — 가 여기에 달려 있습니다.

## 파일과 역할

| 파일 | 역할 | 수명 |
|---|---|---|
| `mistakes.md` | 실수 누적 → 진화 트리거 대기 | **임시** — 진화 완료 시 삭제 |
| `lessons.md` | 교훈 누적 → 규칙/스킬화 대기 | **임시** — 진화 완료 시 삭제 |
| `decisions.md` | 진화 영구 이력 (ADR) | **영속** — 절대 삭제하지 않음 |

> **핵심**: `mistakes.md`와 `lessons.md`는 *진화 대기열*이다.
> 규칙·스킬·hook으로 승급된 항목은 `decisions.md`에 흡수된 뒤 삭제된다.
> 이렇게 하면 메모리 파일이 항상 "아직 진화 안 된 활성 항목"만 담아 작게 유지된다.

## 읽기 프로토콜 (매 세션)
`SessionStart` hook이 위 파일들의 끝부분을 컨텍스트에 자동 출력합니다. 잘렸다고
의심되지 않는 한 수동으로 다시 읽을 필요는 없습니다. 다시 읽는다면 `mistakes.md`를 우선합니다.

## 쓰기 프로토콜

### `mistakes.md`에 언제 기록하나
- 가정이 틀려서 테스트가 실패했을 때.
- 사용자가 교정해 줬을 때.
- 린터/타입 체커/런타임이 거부한 코드를 작성했을 때.
- 자기 수정을 다시 읽고 틀린 걸 발견했을 때.

형식:
```markdown
## YYYY-MM-DDTHH:MM:SSZ — <한 줄 제목>
- 맥락: <어떤 작업 중이었는가>
- 한 일: <틀린 행동>
- 왜 틀렸나: <증상이 아니라 근본 원인>
- 옳은 접근: <다음엔 어떻게>
- 태그: #<영역> #<언어> #<도구>
```

### `lessons.md`에 언제 기록하나
- 직전 실수를 막을 수 있었을, 비자명한 사실을 코드베이스에서 배웠을 때.
- 어디에도 문서화되지 않은 관례가 있을 때.
- 어떤 서브시스템에 함정이 있을 때.

항목은 짧게(1–5줄), 태그를 단다.

### `decisions.md`에 언제 기록하나
- 두 가지 가능한 접근 사이에서 선택했고 그 이유를 기억하고 싶을 때.
- 새로운 패턴/라이브러리/아키텍처 경계를 도입했을 때.

형식:
```markdown
## ADR-NNNN: <제목>
- 날짜: YYYY-MM-DD
- 상태: 제안 | 채택 | ADR-MMMM으로 대체됨
- 기원: mistakes.md #태그 (N회) | lessons.md #태그 | 없음 (직접 결정)
- 맥락: <작용하는 힘들>
- 결정: <무엇을 골랐나>
- 결과: <장점과 단점>
```

> `기원` 필드가 있으면, 해당 mistakes.md/lessons.md 항목은 이 ADR에 흡수되어 삭제된 것이다.

## 메모리 → 진화 연결

메모리는 기록으로 끝나지 않는다. `55-self-evolution.md` 프로토콜에 따라:

- `mistakes.md`에 동일 태그 실수 2회 이상 → 해당 영역의 규칙 강화 또는 hook 생성.
- `lessons.md`에서 반복 패턴 발견 → 스킬로 추출.
- `decisions.md`에서 아키텍처 변경 기록 → `61-architecture.md` 갱신.

`session-start.sh`는 세션 시작 시 반복 실수 태그를 감지하여 진화 필요 알림을
주입하고, `trigger-evolution.sh`는 세션 종료 시 진화 기회를 감지한다.
`evolve` 스킬이 실제 규칙·스킬·hook 생성/보강 절차를 안내한다.

## 승급(진화 완료) 시 삭제 절차

진화가 완료되는 순간 아래 순서로 처리한다:

```
1. decisions.md에 ADR 추가
   - 기원 필드에 삭제할 항목의 태그/날짜 명시
2. mistakes.md 또는 lessons.md에서 해당 항목 삭제
3. 진화 보고 시 "원본 항목 삭제됨" 명시
```

삭제 기준:
- **mistakes.md**: 동일 태그 항목 전체를 규칙/hook으로 승급했을 때 해당 항목 모두 삭제
- **lessons.md**: 스킬/규칙으로 추출했을 때 해당 항목 삭제
- **삭제하지 않는 것**: 아직 진화 조건 미달인 항목, decisions.md

## 위생
- 메모리 파일은 저장소와 함께 커밋된다. 비밀, PII, 고객 식별 데이터 금지.
- mistakes.md와 lessons.md는 승급 후 즉시 삭제로 자연스럽게 작게 유지한다.
  500줄 초과 시엔 승급 대기 항목이 너무 많다는 신호 — 진화를 먼저 돌린다.
- decisions.md는 영속 파일이므로 삭제하지 않는다. 1000줄 초과 시 연도별 분할 고려.
